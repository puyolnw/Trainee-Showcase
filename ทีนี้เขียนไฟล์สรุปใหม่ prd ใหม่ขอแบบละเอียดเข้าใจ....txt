1. Product Summary


The goal is to build a holistic Performance Management System that goes beyond simple appraisals. It supports the full employee lifecycle: goal setting, continuous feedback, 360-degree reviews, and fair scoring via calibration.
Why is this Enterprise-Grade?
* 360 Feedback: Allows peers and matrix managers to provide input.
* Fact-Based: Integrates with HREx Time to show attendance records (Lateness/Absence) directly in the review form, reducing recency bias.
* Calibration: Includes a mechanism for HR to adjust grades across departments to ensure fairness.
________________


2. Integration Architecture


This module does not operate in a silo. It pulls data from existing services to create a "Single Source of Truth."


2.1 Connection with HREx Time


Performance reviews often suffer from subjectivity. We will solve this by pulling objective data from the Time Service.
* Trigger: When a Manager opens a subordinate's Review Form.
* Action: Talent Service calls GET /timesheets/summary?employee_id=... from HREx Time.
* Result: The review form displays a "Discipline Widget":
   * Total Late Arrivals (YTD)
   * Total Absent Days (YTD)
   * Total Sick Leave Used
* Benefit: Managers can score "Discipline/Responsibility" competencies based on actual data, not just memory.

3. Database Schema (The 14 Core Tables)


We have organized the 14 tables into 4 Logical Groups for better understanding.


Group 1: Configuration (Setting the Rules)


These tables define how the review process works before anyone logs in.
#
	Table Name
	Description & Purpose
	1
	PERF_CYCLE
	The Time Window. Defines the start/end dates of the review (e.g., "Year End 2025"). It controls the system status (Setup, Goal Setting, Review, Closed).
	2
	PERF_RATING_SCALE
	The Score Logic. Defines the scoring system (e.g., 1-5, A-E, or "Below/Meets/Exceeds"). Includes the min/max values and label definitions.
	3
	PERF_FORM_TEMPLATE
	The Form Structure. Different roles need different forms. This table links a specific form (e.g., "Manager Form") to a Rating Scale.
	4
	PERF_FORM_SECTION
	The Categories. Divides the form into weighted parts.


(Example: Section 1 = KPIs [70%], Section 2 = Competencies [30%]).
	5
	PERF_COMPETENCY_LIB
	The Question Bank. A library of standard behaviors (e.g., "Leadership," "Communication") linked to Job Levels, so managers don't have to type them from scratch.
	

Group 2: The Transactional Review (The Execution)


These tables store the actual data generated during a review.
#
	Table Name
	Description & Purpose
	6
	PERF_REVIEW
	The Master Header. Represents one employee's review for one cycle. Stores the total_score, final_grade, and current status. Links to Employee & Manager.
	7
	PERF_REVIEW_PARTICIPANT
	The Who's Who (360). Stores everyone involved in a specific review.


(Roles: Self, Primary Manager, Matrix Manager, Peer, Subordinate). Tracks who has submitted their part.
	8
	PERF_ITEM
	The Questions. The specific goals or competencies being evaluated for this person. Contains the weight, target, and actual_result text.
	9
	PERF_ITEM_RATING
	The Scores. Stores the specific score given by a specific participant for a specific item.


Note: One Item can have multiple Ratings (from Self, Manager, Peer).
	

Group 3: Continuous Development (Between Cycles)


These tables are active all year round, not just during review month.
#
	Table Name
	Description & Purpose
	10
	PERF_CHECK_IN
	1-on-1 Notes. Records monthly feedback sessions. Can be marked private (Manager notes only) or shared.
	11
	PERF_IDP
	Growth Plan. (Individual Development Plan). Tracks distinct development activities (e.g., "Take Python Course") separate from performance KPIs.
	

Group 4: Governance & Audit (The "Back Office")


These tables ensure fairness and data integrity.
#
	Table Name
	Description & Purpose
	12
	PERF_CALIBRATION_SESSION
	The Meeting. Records the HR "Calibration" session where managers meet to standardize grades across departments.
	13

4. Core Features & Logic




4.1 Phase 1: Goal Setting


* Employee: Creates PERF_ITEMs (KPIs) in the PERF_REVIEW.
* Validation: System checks if Total Weight = 100%.
* Manager: Approves goals. (Status: GOAL_APPROVED).


4.2 Phase 2: 360 & Self-Assessment


* Nomination: Employee nominates peers. Manager approves the list (creates PERF_REVIEW_PARTICIPANT records).
* Scoring:
   * Peers rate specific Competency sections.
   * Employee rates Self (KPIs + Competencies).


4.3 Phase 3: Manager Evaluation (Time Integration)


* The View: Manager opens the "Scoring Form".
* Time Data Integration: The top of the form fetches data from HREx Time:Attendance Summary (2025):
Lates: 3 times | Absent: 0 days | Sick Leave: 2 days.
* Decision: Manager uses this data to score the "Discipline" competency objectively.
* Calculation: System calculates Weighted Average Score based on Section Weights.


4.4 Phase 4: Calibration & Closure


   * HR Admin: Opens a Calibration Session.
   * Action: HR modifies the final_grade or calibrated_score in the database.
   * Closure: Once finalized, the data is locked and sent to the Payroll Service for bonus calculation.

5. Backend API Specification




5.1 Review Management


   * POST /reviews/init: Start a review cycle for eligible employees.
   * GET /reviews/:id: Get full review details.
   * GET /reviews/:id/attendance-summary: (Integration) Calls Time Service to get Late/Absent stats.


5.2 360 Feedback


   * POST /reviews/:id/participants: Add a peer reviewer.
   * POST /items/:itemId/rate: Submit a score for a specific item.


5.3 Continuous Feedback


   * POST /check-ins: Create a 1-on-1 note.
   * POST /idp: Create a development plan item.

6. Frontend Requirements




6.1 Employee Portal


   * My Performance: Dashboard showing current status.
   * Goal Editor: Dynamic table to add/edit KPIs.
   * Feedback Request: UI to search for colleagues and request 360 feedback.


6.2 Manager Portal


   * Team Dashboard: List of subordinates with status bars (e.g., "Self-Review Pending").
   * Evaluation Form (Split View):
   * Left Side: Employee's Self-Rating & Comments.
   * Right Side: Manager's Input.
   * Top Banner: Attendance Stats (from Time).


6.3 HR Portal


   * Calibration Board: A Scatter Plot or Table view showing distribution of grades (e.g., Bell Curve) allowing drag-and-drop grade adjustment.
